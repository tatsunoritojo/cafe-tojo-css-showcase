G. フォーム見本市

入力部品 20種 — input / checkbox / radio / toggle / バリデーション

フォームは「触って試す」のが一番。実際に入力・クリック・フォーカスしてみてください。つまみで角丸やパディングもライブ調整可能。

フォーム設計の3原則

フォーカス明確

:focus時の見た目を絶対に省略しない。キーボード操作のアクセシビリティに直結。

エラー即時

送信後ではなく、入力中/離脱時にバリデーション。ユーザーの労力を減らす。

タッチ基準

タップ領域 44×44px 以上。スマホで指が届く設計にする。

グループ1: 入力欄のバリエーション

標準 input

padding: 0.6em 0.9em;
border: 1px solid #ccc;
border-radius: 4px;
font-size: 1rem;
width: 100%;

つまみ: 角丸 0(硬め) 〜 12(柔らかめ)。プロダクトの雰囲気で選ぶ

実機: スマホは padding: 0.8em 以上、font-size: 16px 必須（拡大表示防止）

下線のみ（Material Design風）

padding: 0.6em 0;
border: none;
border-bottom: 2px solid #ccc;
background: transparent;
font-size: 1rem;
width: 100%;

つまみ: 下線の太さ 1-3px。太いほど存在感

実機: Material Design、ミニマル系デザインで王道。フォーカス時の色変化必須

丸角（ピル型）

padding: 0.6em 1.2em;
border: 1px solid #ccc;
border-radius: 999px;
font-size: 1rem;
width: 100%;

つまみ: 999px=完全ピル、20-30px=丸角強め

実機: 検索ボックス、SNS系UIで頻出。カジュアルな印象

アイコン付き

/* .wrap */
position: relative;
/* .icn */
position: absolute;
left: 0.8em;
top: 50%;
transform: translateY(-50%);
color: #888;
pointer-events: none;
/* input */
padding-left: 2.4em;

つまみ: アイコン色は#888程度に控えめに。主張しすぎない

実機: 検索・メール・パスワードで定番。アイコンは pointer-events: none でクリック素通り

グループ2: フォーカス表現 クリックしてフォーカスしてみて

outline（青い枠）

input {
  padding: 0.6em 0.9em;
  border: 1px solid #ccc;
  border-radius: 4px;
}
input:focus {
  outline: 3px solid #4a90e2;
  outline-offset: 1px;
  border-color: #4a90e2;
}

つまみ: outline は 2-3px が視認性◯。outline-offset で浮かせる

実機: アクセシビリティ観点で絶対に消さない。デフォルトで十分という立場も有力

枠色 + リング影

input:focus {
  outline: none;
  border-color: #6b4423;
  box-shadow: 0 0 0 3px
    rgba(107, 68, 35, 0.2);
}

つまみ: リングの太さ 2-4px、透明度 0.15-0.3 が適正

実機: Tailwind CSS 流の「リング」。ブランド色と揃えやすい

下線アニメ（展開）

.wrap { position: relative; }
.wrap::after {
  content: '';
  position: absolute;
  left: 0; bottom: 0;
  width: 0; height: 2px;
  background: #6b4423;
  transition: width 0.3s;
}
input:focus + .wrap::after {
  width: 100%;
}

つまみ: transition 0.2-0.4s が気持ちいい速度

実機: Material Design の InputField。「フォーカス中」を視覚で伝える高度UX

背景色変化

input {
  background: #f5f5f5;
  border: none;
  border-radius: 4px;
  padding: 0.7em 0.9em;
  transition: background 0.2s;
}
input:focus {
  outline: none;
  background: #ffffff;
  box-shadow: 0 0 0 2px #6b4423;
}

つまみ: 非フォーカス時を濃く、フォーカスで白く。逆の発想

実機: iOS風の「検索バー」表現。周囲から浮き上がる効果

グループ3: チェックボックス・ラジオ・トグル

標準チェック（accent-color）

label {
  display: inline-flex;
  align-items: center;
  gap: 0.5em;
  cursor: pointer;
}
input[type="checkbox"] {
  accent-color: #6b4423;
  width: 1.2em;
  height: 1.2em;
}

つまみ: accent-color 1プロパティでブランド色に。2021年以降の新定番

実機: OSネイティブ感を保つならこれが最善。カスタマイズは最小限で十分

カスタムチェック（独自デザイン）

input { display: none; }
.check-box {
  width: 20px;
  height: 20px;
  border: 2px solid #6b4423;
  border-radius: 4px;
  display: inline-block;
  position: relative;
}
input:checked + .check-box {
  background: #6b4423;
}
input:checked + .check-box::after {
  content: '✓';
  color: white;
  position: absolute;
  top: -2px;
  left: 3px;
}

つまみ: チェック時に scale/bounce で動きをつけると触感◯

実機: ブランド感重視なら。ネイティブの挙動（キーボード操作）が効くようにaccessibility注意

トグルスイッチ

input { display: none; }
.slider {
  width: 44px;
  height: 24px;
  background: #ccc;
  border-radius: 999px;
  position: relative;
  transition: background 0.25s;
}
.slider::before {
  content: '';
  width: 20px;
  height: 20px;
  background: white;
  border-radius: 50%;
  position: absolute;
  top: 2px;
  left: 2px;
  transition: left 0.25s;
}
input:checked + .slider { background: #6b4423; }
input:checked + .slider::before { left: 22px; }

つまみ: 幅 40-50px / 高さ 22-28px が定番サイズ

実機: 設定ON/OFFで頻出。iOS発祥だが今は全環境で使用

ボタン型ラジオ（セグメント）

input { display: none; }
label span {
  padding: 0.5em 1em;
  border: 1px solid #ccc;
  border-radius: 4px;
  cursor: pointer;
  transition: all 0.2s;
}
input:checked + span {
  background: #6b4423;
  color: white;
  border-color: #6b4423;
}

つまみ: 選択済みは太字+背景濃く、未選択は弱く

実機: サイズ選択、カテゴリ選択で頻出。スマホで親指で押しやすいサイズにする

グループ4: エラー・成功・ヘルプ表示

有効なメール形式で入力

エラー状態（赤枠+メッセージ）

input.error {
  border-color: #c62828;
  background: #fff5f5;
}
input.error:focus {
  box-shadow: 0 0 0 3px
    rgba(198, 40, 40, 0.2);
}
.error-msg {
  color: #c62828;
  font-size: 0.8rem;
  margin-top: 0.3em;
}

つまみ: 赤色は強すぎず #c62828 相当。背景をごく薄いピンクに

実機: 色だけでなくアイコン+テキストも添える（色覚多様性への配慮）

OK

成功状態（緑チェック）

input.success {
  border-color: #2e7d32;
  background: #f1f8f2;
}
.success-msg {
  color: #2e7d32;
  font-size: 0.8rem;
  margin-top: 0.3em;
}

つまみ: 緑は #2e7d32。明るすぎないビリジアン系が上品

実機: リアルタイム検証で即時表示。ログインID重複チェック等で使う

英数記号を組み合わせ、8文字以上で入力

ヘルプテキスト（入力補助）

.label {
  display: block;
  font-weight: bold;
  margin-bottom: 0.3em;
  font-size: 0.9rem;
}
.help-msg {
  color: #888;
  font-size: 0.78rem;
  margin-top: 0.3em;
}

つまみ: ヘルプは #888 程度に控えめに。本文より小さく

実機: 入力ルール、文字数制限、フォーマット指示等に。placeholder の代わりに使う

必須マーク

.required {
  color: #c62828;
  font-size: 0.75rem;
  font-weight: bold;
  margin-left: 0.3em;
}

/* 文字 "必須" と赤星 * の併用がわかりやすい */

つまみ: 赤*だけは不親切。「必須」テキスト併用を推奨

実機: 逆に「任意」を明示する方が実は親切。必須が多いフォームの場合

グループ5: 実用例 — そのまま使える組み合わせ

検索ボックス（クリアボタン付き）

.search-box {
  display: flex;
  align-items: center;
  border: 1px solid #ccc;
  border-radius: 999px;
  padding: 0 0.4em 0 0.9em;
  background: white;
}
.search-box input {
  flex: 1;
  border: none;
  outline: none;
  padding: 0.6em 0.4em;
}
.search-box button {
  background: none;
  border: none;
  cursor: pointer;
  color: #888;
  padding: 0.3em;
}

つまみ: 高さ 36-44px、丸角は pill (999px) が基本

実機: クリアボタンは入力があるときだけ表示（JS制御）が親切

複数入力+ラベル

.form {
  display: flex;
  flex-direction: column;
  gap: 1em;
}
.field {
  display: flex;
  flex-direction: column;
  gap: 0.3em;
}
.label {
  font-weight: bold;
  font-size: 0.9rem;
}

つまみ: フィールド間隔 0.8-1.2em、ラベル→入力間 0.3em

実機: <label for> で紐付け必須。クリック範囲が広がり、アクセシビリティ向上

textarea（複数行）

textarea {
  width: 100%;
  padding: 0.6em 0.9em;
  border: 1px solid #ccc;
  border-radius: 4px;
  font-family: inherit;
  font-size: 1rem;
  resize: vertical;
  min-height: 80px;
}

つまみ: resize: vertical で縦のみリサイズ許可。noneはユーザーの自由を奪う

実機: font-family: inherit で input と揃える。rows で初期高さ指定

ファイル選択（カスタムUI）

input[type="file"] {
  display: none;
}
.file-btn {
  display: inline-flex;
  align-items: center;
  gap: 0.4em;
  padding: 0.6em 1em;
  background: #6b4423;
  color: white;
  border-radius: 4px;
  cursor: pointer;
}
.file-name {
  color: #888;
  margin-left: 0.8em;
  font-size: 0.9rem;
}

つまみ: ブラウザ標準の file input はデザインバラバラ → 必ずカスタム化

実機: ファイル名表示はJSで更新。accept="image/*" で種類制限

教える時のポイント

  • :focus を絶対に省略しない。outline: noneだけは犯罪級。必ず代替の見た目を用意
  • スマホの iOS では input の font-size < 16px で自動拡大される。常に 16px 以上
  • タップ領域は 44×44px 以上。Apple/Googleのガイドライン
  • <label for="id"> と <input id="id"> の紐付けで、ラベルクリック→入力フォーカス。アクセシビリティの基本
  • accent-color（2021年以降）で checkbox/radio をブランド色に1行で変えられる
  • エラー表示は色だけに頼らない。アイコン・テキストを併用（色覚多様性配慮）
  • :invalid / :valid 疑似クラスでバリデーション結果を純CSSで表現可能
  • placeholder は入力ヒント専用。ラベルの代わりにはしない（入力後消えて困る）

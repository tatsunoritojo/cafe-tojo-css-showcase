色選びの3原則

限定する

基本色 + アクセント + ニュートラルの3系統で足りる。多すぎはノイズ。

対比を確保

本文と背景は最低 4.5:1 のコントラスト比。WCAG AA の基準。

変数化する

色は --primary, --bg 等で CSS 変数に。テーマ切替が楽になる。

グループ2: 線形グラデーション

横方向（to right）

background: linear-gradient(
  to right,
  #6b4423,
  #a56b3a
);

つまみ: to right は 90deg と同じ。キーワードか角度かは好みで

実機: ヒーロー背景、ボタン、プログレスバーで頻出。左→右＝進行の方向

縦方向（to bottom）

background: linear-gradient(
  to bottom,
  #8a5a30,
  #4a2e18
);

つまみ: 上=明、下=暗 にすると重力のある自然な立体感

実機: ヒーロー下部のフェード、フッター、モバイルのドロワー背景等

斜め（135deg）

background: linear-gradient(
  135deg,
  #c44,
  #6b4423
);

つまみ: 角度 0-360deg。0=下向き、90=右、180=上、270=左

実機: 45/135deg が最も使用頻度高。動きを感じさせる斜め

3色グラデ（停止位置指定）

background: linear-gradient(
  to right,
  #6b4423,
  #ffd700 50%,
  #c44
);

つまみ: 中間色の停止位置 10-90%。偏らせると劇的に印象が変わる

実機: ロゴ背景、特設ページ見出しで映える。実務ページでは控えめに

グループ3: 特殊グラデーション

放射状（radial-gradient）

background: radial-gradient(
  circle at center,
  #ffd700,
  #6b4423
);

つまみ: at top left等で中心位置を変更。ellipseで楕円

実機: スポットライト効果、ボタンのホバー光沢で活躍

円錐（conic-gradient）

background: conic-gradient(
  from 0deg at center,
  #c44, #ffd700,
  #2e7d32, #1565c0,
  #c44
);

つまみ: from で開始角度。色を循環させると放射状パイチャート風

実機: ローディング進捗、カラーピッカー、パイチャートに応用可能

虹色グラデ

background: linear-gradient(
  to right,
  #e53935, #fb8c00, #fdd835,
  #43a047, #1e88e5, #5e35b1
);

つまみ: 色の数だけ継ぎ目が増える。6-7色が自然

実機: Pride/多様性の文脈、音楽アプリ等で象徴的に使われる

透明フェード（色 → 透明）

background: linear-gradient(
  to right,
  rgba(107, 68, 35, 0.8),
  transparent
);

つまみ: rgbaの透明度 0-1。画像の上に重ねると文字が読みやすくなる

実機: ヒーロー画像のオーバーレイ、スクロールフェードで必須のテクニック

グループ4: 背景パターン — CSSだけで絵柄を作る

縞模様（repeating-linear）

background: repeating-linear-gradient(
  0deg,
  #6b4423,
  #6b4423 12px,
  #8a5a30 12px,
  #8a5a30 24px
);

つまみ: 角度 0-180deg。0=横縞、90=縦縞、45=斜め

実機: 工事中/警告、ローディングバー、プリント風意匠で頻出

斜めストライプ（細め）

background: repeating-linear-gradient(
  45deg,
  #f5efe6,
  #f5efe6 6px,
  #e5d9c7 6px,
  #e5d9c7 12px
);

つまみ: 幅を揃えると正方形、変えると非対称。明度差は小さいほど上品

実機: ブランド背景、壁紙代わりに。濃い色は目がチカチカするので淡色で

ドット（radial）

background-image: radial-gradient(
  circle, #6b4423 2px, transparent 2px
);
background-size: 24px 24px;

つまみ: background-size で格子の間隔。広げるとまばらに

実機: 紙物印刷風の背景、メモ帳UIで使える。ドットサイズを小さく保つのが上品

グリッド線

background-image:
  linear-gradient(#e5d9c7 1px, transparent 1px),
  linear-gradient(90deg, #e5d9c7 1px, transparent 1px);
background-size: 20px 20px;
background-color: #fdfaf4;

つまみ: マス目のサイズ。細かいほど方眼紙感、粗いほど建築図面感

実機: 設計系UI、メモアプリ、Figmaライクな背景で活躍

グループ5: 応用エフェクト

くっきり文字

ぼかし背景（filter: blur）

/* .blur-bg */
background: linear-gradient(135deg, #c44, #6b4423, #ffd700);
filter: blur(8px);
position: absolute;
inset: 0;

つまみ: blur 0-30px。文字の後ろに敷くと読みやすくなる

実機: ヒーロー背景、ローディング時の画像マスクで使う

Glass

ガラスモーフィズム

/* .glass-card */
background: rgba(255, 255, 255, 0.3);
backdrop-filter: blur(10px);
-webkit-backdrop-filter: blur(10px);
border: 1px solid rgba(255, 255, 255, 0.4);
border-radius: 12px;

つまみ: 透過度 0.1-0.5 が適正。低すぎると見えない、高すぎると普通の白箱に

実機: iOS風UI、映えるダッシュボードに。背景が複雑でないと効果が薄い

光沢

光沢アニメーション

position: relative;
overflow: hidden;
background: #6b4423;

/* ::before で斜めの光を流す */
/* background: linear-gradient(
     120deg, transparent,
     rgba(255,255,255,0.4), transparent); */

つまみ: アニメーション秒数 2-4s が上品。高速だと安っぽくなる

実機: プレミアム感の演出、スケルトンローディングで活用

Mixable CSS

グラデーション文字

background: linear-gradient(
  135deg, #c44, #ffd700, #6b4423);
background-clip: text;
-webkit-background-clip: text;
color: transparent;
font-weight: bold;

つまみ: グラデ色の数と角度を変えれば無限バリエ。文字色transparent必須

実機: ロゴ、ヒーロー見出しで映える。本文には使わない（可読性低下）

教える時のポイント

  • 色は変数化する — --primary, --bg 等の CSS 変数で定義。テーマ切替・保守が段違いに楽
  • コントラスト比 4.5:1（本文）・3:1（大文字）を守る。WCAG AA が業界標準
  • linear-gradient の角度は to right キーワードか 90deg 数値。キーワードは直感的、数値は知識必要
  • repeating-*-gradient で CSSだけで絵柄 を作れる。画像不要でファイルサイズ圧縮
  • backdrop-filter は iOS/macOS 風のガラス表現。古い Android で非対応なのでフォールバック背景を用意
  • グラデ色は 3色以下 が上品。4色以上は「虹」か「宇宙」の印象になる
  • background-clip: text 系は -webkit- プレフィックスも併記が無難

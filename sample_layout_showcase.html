実務で読むための3つの視点

つまみ

どの値を変えると何が動くか。実務はほぼ「既存CSSの値を調整」作業。

実機

PC で見て完成でも、スマホで崩れる。ブレイクポイント前提の書き方を身につける。

第一選択

同じ結果でも今どきの実務では auto-fit + minmax()。レガシー保守で margin: auto。

グループ1: Flexbox 基本 — 方向と間隔

A B C

横並び（flex row）

display: flex;
gap: 0.5em;

つまみ: gap を 0.3em→2em で密度変更。子に flex-shrink: 0 で縮小防止

実機: 480px 以下では flex-direction: column に切替が定石

A B C

縦並び（flex column）

display: flex;
flex-direction: column;
gap: 0.5em;

つまみ: gap で行間。order: -1 で特定要素を先頭に移動

実機: モバイルファーストの基準形。PC で @media (min-width: 768px) { flex-direction: row }

A B C

広めのギャップ

display: flex;
gap: 1.5em;

つまみ: 0.5em(詰) / 1em(標準) / 2em(ゆったり) の3段階が実務基準

実機: デバイス幅で可変に → gap: clamp(0.5em, 2vw, 2em)

A B C D E F

折り返し（wrap）

display: flex;
flex-wrap: wrap;
gap: 0.5em;

つまみ: 子に flex: 1 1 200px で最小200pxの可変カード列に

実機: 折り返し前提なら Grid auto-fit の方が列数制御が楽（G3-10参照）

グループ2: Flexbox 配置 — 主軸／交差軸

A B C

中央揃え

display: flex;
justify-content: center;
align-items: center;
gap: 0.5em;

つまみ: 親の height or min-height が必要。vh で画面全体中央に

実機: ヒーロー領域は min-height: 80vh が相場。スマホ 100vh は下部UIで切れるので避ける

A B C

両端配置（space-between）

display: flex;
justify-content: space-between;
align-items: center;

つまみ: 子が奇数だと中央要素がそのまま中央に残る。左右2要素前提で使うのが安全

実機: ヘッダーの最頻出形。スマホでは右側をハンバーガー1個に集約

A B C

均等配置（space-around）

display: flex;
justify-content: space-around;
align-items: center;

つまみ: space-evenly なら両端余白も等しい（2020年以降主流）

実機: タブバー、下部ナビで頻出。5個以上並ぶとスマホで窮屈 → 横スクロール化

A B C

右寄せ（flex-end）

display: flex;
justify-content: flex-end;
gap: 0.5em;

つまみ: 特定の1子だけ右寄せなら子に margin-left: auto が定石

実機: フォーム送信ボタン、モーダル OK/Cancel の配置で頻出

グループ3: Grid 基本 — 列と行の指定

A B C

3等分グリッド

display: grid;
grid-template-columns: repeat(3, 1fr);
gap: 0.5em;

つまみ: repeat(3, 1fr) で3等分。固定+可変混在なら 200px 1fr 1fr

実機: スマホで潰れる → @media (max-width: 600px) { grid-template-columns: 1fr } 必須

A B C D

自動フィット（レスポンシブ）

display: grid;
grid-template-columns:
  repeat(auto-fit, minmax(60px, 1fr));
gap: 0.5em;

つまみ: minmax(240px, 1fr) の 240px が1列の最小幅。商品カードなら 240-320px

実務の第一選択: メディアクエリ不要で全デバイス対応。迷ったらこれ

A B

サイドバー（固定＋可変）

display: grid;
grid-template-columns: 60px 1fr;
gap: 0.5em;

つまみ: 60px を 240px 等に。右サイドなら 1fr 240px

実機: スマホで縦積み必須 → @media (max-width: 768px) { grid-template-columns: 1fr }

A B C D

2×2 グリッド

display: grid;
grid-template-columns: 1fr 1fr;
grid-template-rows: auto auto;
gap: 0.5em;

つまみ: 1要素を縦2段に伸ばすなら子に grid-row: span 2

実機: ダッシュボードで頻出。スマホでは grid-template-columns: 1fr で1列化

グループ4: よくある全体レイアウト

Header Main Footer

ヘッダー/メイン/フッター

display: grid;
grid-template-rows: auto 1fr auto;
gap: 0.3em;
min-height: 100vh;

つまみ: 1fr が可変領域。min-height: 100vh で全画面カバー

実機: 全デバイス共通の骨格。スマホ時は 100dvh（動的vh）推奨でアドレスバー問題回避

Side Main

サイドバー + メイン

display: grid;
grid-template-columns: 1fr 3fr;
gap: 0.5em;
min-height: 100%;

つまみ: 1fr 3fr の比率変更。固定幅なら 240px 1fr

実機: 管理画面の定番。スマホではサイドバーをドロワー/オフキャンバス化が必須

1 2 3 4 5 6

カードグリッド

display: grid;
grid-template-columns:
  repeat(auto-fill, minmax(50px, 1fr));
gap: 0.4em;

つまみ: minmax の最小値で列数が決まる（120px→多数列、320px→少数列）

実機: 商品/記事一覧で必須。実務では 240-320px が現実解。auto-fill vs auto-fit は空列の扱いが違う

H S M

名前付きエリア

display: grid;
grid-template-columns: 1fr 2fr;
grid-template-areas:
  "h h"
  "s m";
gap: 0.3em;

つまみ: "h h" のような文字列で構造が一目瞭然。エリア名を変えれば順序も変わる

実機: スマホ用に grid-template-areas: "h" "m" "s" と上書きで縦積みへ切替可能

グループ5: センタリング技法 — 4つの流派を比較

中

Flex で中央

display: flex;
justify-content: center;
align-items: center;

つまみ: gap で複数子要素の中央配置も可能。親の高さが必要

実務の第一選択: 2024年以降の標準。全ブラウザ対応で迷う要素なし

中

Grid で中央（place-items）

display: grid;
place-items: center;

つまみ: place-items = align-items + justify-items の短縮形

実機: 最短1行。Safari 14 未満で非対応のため古い端末サポート時は Flex 版を選ぶ

中

absolute + transform

position: relative;
/* 子要素:
   position: absolute;
   top: 50%;
   left: 50%;
   transform: translate(-50%, -50%); */

つまみ: 親に position: relative 必須。子の幅/高さ未知でも中央に置ける

実機: モーダル、ツールチップで現役。Flex/Grid が使えない特殊環境の最終手段

中

margin: auto（水平のみ）

display: block;
/* 子要素:
   margin: 0 auto;
   width: fit-content; */

つまみ: width 指定必須（auto では効かない）。width: fit-content で可変幅に対応

実機: メールHTML（Flex/Grid 非サポート）、レガシーコード保守で頻出

レスポンシブ設計の判断フロー

  1. auto-fit + minmax() で解決できないか をまず検討（G3-10 / G4-15）。これで済むなら メディアクエリ不要
  2. 固定+可変の明確な2カラム構造なら Grid の サイドバー型（G3-11）＋スマホ時 1fr に切替
  3. 1方向の並びのみなら Flex row → スマホで column の定石（G1-1 / G1-2）
  4. 全体骨格は Holy Grail（G4-13） で min-height: 100dvh が 2024 年以降のベストプラクティス
  5. センタリングは Flex or Grid place-items（G5-17 / G5-18）。それ以外は保守・特殊用途のみ

教える時のポイント

  • 「つまみ」思考 — 実務はゼロから書くより既存の値を調整するほうが圧倒的に多い。どの値が可変かを意識する
  • モバイルファースト — スマホ用CSSを基準にし、@media (min-width: 768px) で PC 用を上書きする書き方が今どき
  • Flexbox は「1方向」、Grid は「2次元」 — 選び方の第一基準
  • gap は Flex でも Grid でも使える。margin より圧倒的に楽
  • レイアウトは 親のCSS。子側に書いても動かない、という混乱が初学者の8割の詰まりどころ
  • レスポンシブの第一選択は auto-fit + minmax()。メディアクエリはその後
  • dvh / svh / lvh（2024年主流）はスマホのアドレスバー問題を解決する新単位

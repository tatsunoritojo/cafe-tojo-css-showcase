パラメータ早見表

構文上の位置 意味 例（値）
1つ目: アニメ名 どのキーフレーム定義を使うか breathe, typing 等（@keyframesの名前）
2つ目: 継続時間 1サイクルに何秒/ミリ秒かけるか 0.5s 速い / 3s 標準 / 8s ゆっくり
3つ目: タイミング関数 速度の緩急の付き方 linear 等速 / ease 自然 / ease-in 加速 / ease-out 減速 / steps(N) カクカク
4つ目: 遅延（任意） 開始までの待ち時間 0s 即座 / 1s 1秒後 / -1s 途中から
5つ目: 繰り返し回数 何回実行するか 1 1回 / 3 3回 / infinite 無限
6つ目: 方向 終わったら逆再生するか normal 毎回0→100 / alternate 0→100→0（往復）

省略記法と個別記法

/* 一括指定（ショートハンド） */
animation: breathe 3s ease-in-out 0s infinite alternate;
         /* 名前   時間  緩急      遅延 回数     方向    */

/* 分割指定（個別に書きたい時） */
animation-name: breathe;
animation-duration: 3s;
animation-timing-function: ease-in-out;
animation-iteration-count: infinite;
animation-direction: alternate;

グラデーションの流れる速度の仕組み

グラデ文字は background-size も効きます:

.txt-flow {
  background: linear-gradient(...);
  background-size: 300% auto;   /* ← ここが大きいほど、animation時間が同じでも */
  /*                               移動距離が長くなり、効果の"速さ"感が変わる */
  animation: flowGradient 3s linear infinite;
}
@keyframes flowGradient {
  from { background-position: 0% center; }
  to   { background-position: 300% center; }
                    /* ↑ background-size と同じ値にすると、ちょうど1周する */
}

ちょっと高度: ホバー時だけ高速に

.txt-breathe         { animation: breathe 3s infinite; }
.txt-breathe:hover   { animation-duration: 0.5s; }
/* ↑ プロパティを個別に上書きできる */

教える時のポイント

  • 文字装飾の大半は color / text-shadow / background-clip の組み合わせで実現できる
  • 多重 text-shadow は発光・3D・浮き彫りなど幅広い表現の核心
  • グラデ文字は background-clip: text + color: transparent の2点セット
  • アニメーションは @keyframes + animation プロパティ
  • **やりすぎ注意**: 装飾は主張が強いので1画面に1〜2個が原則

アニメーション3原則

速度

UI反応 = 0.15-0.3s、登場 = 0.3-0.8s、注意喚起 = 1-2s。遅すぎると重く、速すぎると見逃す。

イージング

ease-out（速→遅）が自然。linearは機械的、cubic-bezierでこだわり。

配慮

@media (prefers-reduced-motion) でアニメ不要ユーザーに配慮。実務で忘れがち。

グループ1: 登場アニメ（infinite alternate でデモ表示中）

カフェ東城

フェードイン

@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}
.element {
  animation: fadeIn 1.5s ease-out;
}

/* デモでは alternate infinite */
animation-duration: 1.5s;

つまみ: 0.3-0.8s が自然。長すぎると焦れったい

実機: スクロール追従で要素を段階的に表示するライブラリ（AOS等）と組み合わせるのが定番

カフェ東城

下からスライドイン

@keyframes slideUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
.element {
  animation: slideUp 0.6s ease-out;
}

つまみ: translateY は 10-40px が自然。大きすぎると「飛んでくる」感

実機: 登場の王道。モーダルやトーストにも応用可

カフェ東城

スケールイン（ポップ）

@keyframes scaleIn {
  from { transform: scale(0.5); opacity: 0; }
  to   { transform: scale(1);   opacity: 1; }
}
.element {
  animation: scaleIn 1.5s
    cubic-bezier(0.25, 1.5, 0.5, 1);
}

つまみ: cubic-bezier で弾む動き。1.5 部分を大きくすると跳ねる

実機: モーダル登場、バッジ追加時の通知等に

カフェ東城

回転イン

@keyframes rotateIn {
  from {
    transform: rotate(-90deg) scale(0);
    opacity: 0;
  }
  to {
    transform: rotate(0) scale(1);
    opacity: 1;
  }
}

つまみ: 回転角度 -90〜-180deg。派手すぎは禁物

実機: アイコンの登場、達成バッジ等のご褒美演出に

グループ2: ホバー応答 マウスを乗せてみて

拡大（scale）

button {
  transition: transform 0.2s ease;
}
button:hover {
  transform: scale(1.1);
}

つまみ: scale(1.05-1.15)。1.2以上だと「触った感」を超えて違和感

実機: ボタン、カード、サムネで王道。スマホでは :active にも効かせる

回転

.icon {
  transition: transform 0.4s ease;
}
.icon:hover {
  transform: rotate(90deg);
}

つまみ: 90deg(直角) / 180deg(反転) / 360deg(一周) で印象変化

実機: 設定アイコン、リロードボタン等の"回るもの"に限定

色変化

button {
  background: #6b4423;
  color: white;
  transition: background-color 0.25s ease;
}
button:hover {
  background: #c44;
}

つまみ: 色相変化より明度変化が自然。同系色で濃く/薄く

実機: CTAボタンの定番。トランジションは 0.15-0.3s が読み心地良い

影が濃く

影強調

.card {
  box-shadow: 0 2px 6px rgba(0,0,0,0.1);
  transition: box-shadow 0.3s ease, transform 0.3s ease;
}
.card:hover {
  box-shadow: 0 10px 24px rgba(0,0,0,0.18);
  transform: translateY(-3px);
}

つまみ: blur を 3-4 倍、透明度を 2 倍。translateY-2〜-4px を併用

実機: カードUIの選択可能感。クリッカブル表現の基本

グループ3: 注意喚起（無限ループ）

パルス（拡縮呼吸）

--pulse-scale: 1.1;

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%      { transform:
    scale(var(--pulse-scale)); }
}
.element {
  animation: pulse 1.5s ease-in-out infinite;
}

つまみ: 1.05-1.15 が自然。大きいほど「目立て」アピール強め

実機: 通知バッジ、CTAボタン、新着ラベルで定番

シェイク（左右振動）

--shake-x: 8px;

@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25% { transform:
    translateX(calc(var(--shake-x) * -1)); }
  75% { transform:
    translateX(var(--shake-x)); }
}
.element {
  animation: shake 0.4s ease-in-out 3;
}

つまみ: 4-10px。大きいほど激しい怒り風

実機: フォームバリデーション失敗で定番。3回限定が自然

バウンス（跳ねる）

--bounce-h: 16px;

@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50%      { transform:
    translateY(calc(var(--bounce-h) * -1)); }
}
.element {
  animation: bounce 1s ease-in-out infinite;
}

つまみ: 10-25px が自然。大きすぎると「落ち着きがない」印象

実機: 「スクロールしてね」の矢印、ゲーム風UIで

録画中

点滅

@keyframes blink {
  50% { opacity: 0.2; }
}
.element {
  animation: blink 1s steps(2) infinite;
  color: #c44;
}

つまみ: steps(2) でカクカク点滅、ease-in-out でなだらか

実機: 録画中、リアルタイム通信、緊急性の表現。頻度は低く抑える（うるさい）

グループ4: ローディング表現

スピナー（円形）

@keyframes spin {
  to { transform: rotate(360deg); }
}
.spinner {
  width: 32px;
  height: 32px;
  border: 3px solid #e5d9c7;
  border-top-color: #6b4423;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}

つまみ: 0.6-1s が定番。遅いと「止まってる?」と感じる

実機: データ読込中の標準表現。SVGスピナーより CSS の方が軽量

ドット3点（順次点滅）

@keyframes dotBlink {
  0%, 80%, 100% { opacity: 0.3; }
  40%           { opacity: 1; }
}
.dots span {
  animation: dotBlink 1.4s infinite;
}
.dots span:nth-child(2) { animation-delay: 0.2s; }
.dots span:nth-child(3) { animation-delay: 0.4s; }

つまみ: animation-delay を 0.2s ずつずらして波の表現

実機: チャットの「入力中...」表示でお馴染み

プログレスバー（連続流れ）

@keyframes progress {
  from { background-position: 0 0; }
  to   { background-position: 40px 0; }
}
.bar {
  height: 10px;
  background: repeating-linear-gradient(
    45deg, #6b4423 0, #6b4423 10px,
    #8a5a30 10px, #8a5a30 20px);
  animation: progress 2s linear infinite;
}

つまみ: 1-3s が自然。速すぎるとストライプが流れ過ぎる

実機: 進行度不明のロード時に。%付きバーには使わない

スケルトン（骨格ロード）

@keyframes skel {
  0%   { background-position: -200% 0; }
  100% { background-position: 200% 0; }
}
.skel-line {
  height: 12px;
  border-radius: 4px;
  background: linear-gradient(90deg,
    #eee 25%, #f5f5f5 50%, #eee 75%);
  background-size: 200% 100%;
  animation: skel 1.5s infinite;
}

つまみ: ハイライトの明度を少しだけ変える（#eee→#f5f5f5）

実機: SNS系アプリの定番。実データの形と合わせるのがコツ

グループ5: 応用

Mixable CSS

タイピング風

@keyframes typing {
  from { width: 0; }
  to   { width: 11ch; }
}
@keyframes caret {
  50% { border-color: transparent; }
}
.text {
  font-family: monospace;
  overflow: hidden;
  white-space: nowrap;
  border-right: 2px solid #6b4423;
  animation: typing 2s steps(11) infinite,
             caret 0.5s step-end infinite;
}

つまみ: steps(N) の N は文字数と合わせる。ch 単位で幅指定

実機: ヒーロー見出し、ポートフォリオで映える。長文には使わない

カードフリップ（ホバー）

.card {
  perspective: 600px;
}
.inner {
  transform-style: preserve-3d;
  transition: transform 0.6s;
}
.card:hover .inner {
  transform: rotateY(180deg);
}

  position: absolute;
  backface-visibility: hidden;
}
.back {
  transform: rotateY(180deg);
}

つまみ: perspective 400-800px で遠近感。transition 0.5-0.8s

実機: FAQの回答表示、ゲームUI、プロフィールカード裏面

無限回転

@keyframes spinLoop {
  to { transform: rotate(360deg); }
}
.icon {
  animation: spinLoop 4s linear infinite;
}

つまみ: 遅い(5-10s) = 背景装飾、速い(1-2s) = ロード感

実機: 歯車/地球儀/ディスク の象徴表現、背景のアクセント

Mixable

波打ち（文字別遅延）

@keyframes wave {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-10px); }
}
.wave span {
  display: inline-block;
  animation: wave 1.2s ease-in-out infinite;
}
.wave span:nth-child(2) { animation-delay: 0.1s; }
.wave span:nth-child(3) { animation-delay: 0.2s; }
/* ...以降も 0.1s ずつずらす */

つまみ: 遅延の間隔 0.08-0.15s。揃えるとウェーブ、不揃いだとランダム感

実機: ロゴアニメ、ゲームのHPバー、音楽アプリのイコライザ表現に

教える時のポイント

  • 動かす前に「なぜ動かすか」 — 注意喚起? 状態変化? 装飾? 目的不明なアニメはノイズ
  • transition は「変化のルール」、animation は「キーフレームで定義された動き」
  • 実務の速度基準: UI反応 0.15-0.3s、登場 0.3-0.8s、注意喚起 1-2s
  • イージングは ease-out が8割。linearは時計やロード等の機械的な動きに限る
  • パフォーマンス: transform と opacity のみがGPUアクセラレートされる。width/top 等は避ける
  • アクセシビリティ: @media (prefers-reduced-motion: reduce) { animation: none; } を忘れずに
  • CSS変数を @keyframes 内で使えば、アニメの挙動をインスタンスごとに変えられる
